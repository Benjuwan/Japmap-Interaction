# Japmap-Interaction
SVG画像内の各種オブジェクトをグループ（`g`）ごとに分割し、それぞれにインタラクション（ページ内ジャンプ）を付けたUIです。  
デモとして日本地図を設定しています。
 
> **前提**: インタラクション対象のSVGは、エリアごとに意味のある `id` が付与された状態で用意する必要があります。  
> 詳細は [SVGの準備について](#svgの準備について) を参照してください。

## SVGの準備について
本UIを機能させるには、**エリアごとに `id` が付与されたSVGファイル**が必要です。  
`id` のないSVG（座標の羅列のみのもの）はAIによる自動変換が困難なため、事前の整備が必須です。
 
### 推奨フロー
 
```
1. フリー配布のSVGマップまたはAiデータを入手する
       ↓
2. Adobe Illustrator などで SVG を開き、各エリアのレイヤー名を命名規則に従って設定する
       ↓
3. SVGとして書き出す
       ↓
4. 書き出したSVGと後述のプロンプトをAIに渡し、Reactコンポーネントを生成する
```
 
### SVG入手先の例
 
- [国土数値情報ダウンロードサービス](https://nlftp.mlit.go.jp/ksj/)（行政区域データなど）
- [白地図専門店](https://www.freemap.jp/)
### id 命名規則
 
エリアのレイヤー名（= SVGの `id`）は、以下の命名規則に従って設定してください。  
AIがエリアの粒度を正しく認識するために使用します。
 
| プレフィックス | 粒度 | 例 |
|---|---|---|
| `area-` | 地方単位 | `area-kinki` |
| `pref-` | 都道府県単位 | `pref-osaka` |
| `city-` | 市区町村単位 | `city-namba` |
| `unit-` | 上記に当てはまらないその他の区分 | `unit-custom` |
 
> **注意**: `id` が付いていないSVGの場合、AIによる自動エリア分けは困難です。  
> その場合はAdobe Illustratorなどのソフトでレイヤー名を整備してから再度書き出してください。
 
## AIでReactコンポーネントを生成する
 
整備済みのSVGコードと以下のプロンプトをAIに渡すことで、インタラクション付きのReactコンポーネントを生成できます。
 
### プロンプト
 
````markdown
## タスク： マップ画像（SVGファイル）のエリア分けとインタラクションUIの実装
ユーザーから渡されたマップ画像（SVGコード）を確認し、インタラクション可能なReactコンポーネント（JSX）に変換してください。
 
### 前提・背景情報
ユーザーは「用意したマップ画像の特定のエリアをクリックすると、ページ内ジャンプや特定の処理を実行するUI」を実装しようとしています。
そのためには、SVGの各エリア（`<g>` または `<path>`）に対してインタラクション関数を付与する必要があります。
 
### 実行手順とルール
ユーザーから渡されたSVGコードに対し、以下のルールに従って属性を付与・修正してください。
 
#### 1. 対象の特定とラップ（最重要）
意味的なまとまりを持つタグ（例: `id="tokyo"` などのIDが付いた `<g>`, `<path>`, `<polygon>` 等）を対象とします。  
**インタラクションを付与する際は、対象の図形要素に直接属性を付けるのではなく、必ず新規の `<g>` タグで対象要素をラップ（包む）し、その `<g>` タグに対して以下の属性を付与**してください。
 
> [!NOTE]
> 既存の要素に id が付いた `<g>` タグがある場合は、その `<g>` を新規 `<g>` でラップするのではなく、その既存の `<g>` 自体に直接インタラクション属性を追加してください。  
> 新規ラップが必要なのは `<path>`, `<polygon>`, `<rect>` 等の単体図形タグに直接 ID が付いている場合のみです。
 
#### 2. 属性の追加・変更（ラップした `<g>` に対して）
- **注意**: クリック判定を確実にするため、対象の `<g>` タグには必ず `style={{ pointerEvents: "all" }}` を付与してください。（親要素の `pointerEvents: "none"` を継承してクリック不可になる事故を防ぐためです）
- `data-target` 属性を付与。命名規則（優先順位）は既存のID等を参考に `area-` > `pref-` > `city-` > `unit-` をプレフィックスしてください。
  - `area-`：地方単位（例: area-kinki）
  - `pref-`：都道府県単位（例: pref-osaka）
  - `city-`：市区町村単位（例: city-namba）
  - `unit-`：上記に当てはまらないその他の区分
- `onClick={(e) => clickInteractionArea(e)}` を付与してください。
- Tailwind CSS のクラス `className="cursor-pointer duration-250 hover:opacity-[.5] transition-all"` を付与してください。（※ `class` ではなく `className` を使用すること）
 
#### 3. 元の描画データの保持
`d` や `points` 属性などのパスデータ、`viewBox`、`fill` などの描画に関わる属性は絶対に改変・削除しないでください。  
元の図形要素が持っていたクラス（例: `className="cls-1"`）はそのまま図形要素に残してください。
 
#### 4. 出力形式
純粋なHTMLではなく、Reactでそのまま使用できるJSX/TSX形式で出力してください。
 
### コード変換例（Before / After）
 
【Before】（ユーザーから渡される元のSVGの一部。単体の図形にIDがついているケース）
```xml
<polygon id="osaka-city" class="cls-1" points="396.7 451.02 401.41 480.44..." />
```
 
【After】（あなたが生成すべき出力。新規の `<g>` でラップし、そこにインタラクションを付与する）
```tsx
<g 
  id="map-city-osaka" 
  data-target="city-osaka" 
  onClick={(e) => clickInteractionArea(e)} 
  style={{ pointerEvents: "all" }}
  className="cursor-pointer duration-250 hover:opacity-[.5] transition-all"
>
  <polygon className="cls-1" points="396.7 451.02 401.41 480.44..." />
</g>
```
 
### 制約事項（重要）
- **AIによる視覚的認識の限界**: もし渡されたSVGが「意味のあるID（レイヤー名）が全くない、ただの座標の羅列」であり、エリア分けが推測不可能な場合は、無理に分割せずユーザーに以下のいずれかを提案してください。
  - **A案**: Adobe Illustrator等のソフトでレイヤー名を明示的に設定（例: `pref-osaka` 等）してから再度SVGを書き出してもらう。
  - **B案**: シンプルなグリッド（田の字型の4分割など）で機械的に区切る代替案。
- ユーザーから渡されたデータがSVG形式でない場合は、SVGへの変換が必要な旨を伝えてください。
````
